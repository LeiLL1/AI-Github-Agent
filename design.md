# AI GitHub Agent 设计规范

本规范用于统一当前应用的 GPT Web 对话风格：单列消息流、底部固定输入框、轻量侧边栏，以及与每轮回答绑定的项目推荐卡片。整体视觉以白色和浅灰为底，使用细边框建立层级，深色只用于关键行动、评分和焦点状态。

## 1. 配色

### 基础色

| 类型 | 用途 | 色值 |
| --- | --- | --- |
| 主色 | 主按钮、评分、关键行动 | `#111111` |
| 主色 Hover | 主按钮 hover、链接 hover | `#000000` |
| 主色浅底 | 用户消息、选中态浅背景 | `#F4F4F4` |
| 页面背景 | 主内容区背景 | `#FFFFFF` |
| 主卡片背景 | 卡片、指标、输入面板 | `#FFFFFF` |
| 次级浅底 | Chip、轻量标签、弱化工具条 | `#F7F7F7` |

### 文字色

| 类型 | 用途 | 色值 |
| --- | --- | --- |
| 标题文字 | 页面标题、Hero 标题、核心名称 | `#0D0D0D` |
| 正文文字 | 正文、指标值、主要说明 | `#171717` |
| 次级文字 | 描述、说明、卡片摘要 | `#424242` |
| 弱提示文字 | Caption、元信息、时间、辅助说明 | `#6B6B6B` |
| 反白文字 | 深色主按钮文字 | `#FFFFFF` |

### 边框和状态色

| 类型 | 用途 | 色值 |
| --- | --- | --- |
| 默认边框 | 卡片、Metric、Hero 面板 | `#E5E5E5` |
| 弱边框 | Chip、轻量分隔 | `#EDEDED` |
| 聚焦描边 | 输入框、按钮 focus ring | `rgba(0, 0, 0, 0.12)` |
| 成功色 | 成功提示、收藏完成 | `#12B76A` |
| 警告色 | 限流等待、配置提醒 | `#F79009` |
| 错误色 | GitHub/LLM 错误提示 | `#D92D20` |
| 错误浅底 | 错误消息背景 | `#FEF3F2` |

## 2. 字体

### 字体族

优先使用系统无衬线字体，保持接近 Grok Web 的清爽工具感。

```css
font-family: Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

### 字号、字重和行高

| 场景 | 字号 | 字重 | 行高 |
| --- | --- | --- | --- |
| Hero 标题 | `2.05rem` / 约 `33px` | `760` | `1.15` |
| 移动端 Hero 标题 | `1.65rem` / 约 `26px` | `760` | `1.18` |
| 页面标题 | `1.5rem` / `24px` | `700` | `1.25` |
| 模块标题 | `1.05rem` / 约 `17px` | `750` | `1.35` |
| 正文 | `0.91rem` / 约 `15px` | `400` | `1.55` |
| 辅助说明 | `0.9rem` / 约 `14px` | `400` | `1.45` |
| Section Label | `0.84rem` / 约 `13px` | `650` | `1.3` |
| 元信息 / Chip | `0.82rem` / 约 `13px` | `400` | `1.25` |
| 极弱标签 | `0.76rem` / 约 `12px` | `400` | `1.2` |
| 指标值 | `1rem` / `16px` | `700` | `1.25` |

规则：

- 标题保持短、重、紧凑，正文保持可读行高。
- 不使用负字距，不随视口宽度缩放字号。
- 卡片内部标题不要使用 Hero 级字号，避免破坏工具型界面的密度。

## 3. 间距

### 页面布局

| 场景 | 数值 |
| --- | --- |
| 内容最大宽度 | `1280px` |
| 桌面左右页边距 | `3rem` / `48px` |
| 移动端左右页边距 | `1rem` / `16px` |
| 页面顶部间距 | `1.2rem` / 约 `19px` |
| 页面底部间距 | `4.5rem` / `72px` |

### 组件间距

| 场景 | 数值 |
| --- | --- |
| Hero 面板内边距 | `1.15rem 1.25rem` |
| 卡片内边距 | `0.95rem 1rem` |
| 指标卡内边距 | `0.7rem` |
| Stat Pill 内边距 | `0.65rem 0.75rem` |
| Answer Summary 内边距 | `0.8rem 0.95rem` |
| Chip 内边距 | `0.18rem 0.55rem` |
| 卡片上下间距 | `0.65rem 0` |
| Section Label 外边距 | `0.75rem 0 0.35rem` |
| Hero 底部间距 | `1rem` |
| 元信息横向 gap | `0.45rem` |
| Stat Grid gap | `0.65rem` |

### 间距使用规则

- 页面级模块之间使用 `16px` 到 `24px`。
- 卡片列表之间使用 `10px` 到 `12px`，保持可扫描密度。
- 同一组件内部相邻元素使用 `4px`、`8px`、`12px` 的递进。
- 工具按钮和筛选控件优先保持紧凑，不使用营销页式大留白。

## 4. 组件样式

### 按钮

| 属性 | 规则 |
| --- | --- |
| 高度 | 最小高度 `2.35rem` / 约 `38px` |
| 圆角 | `9px` |
| 主按钮背景 | `#111111` |
| 主按钮文字 | `#FFFFFF` |
| 次级按钮背景 | `#FFFFFF` |
| 次级按钮边框 | `1px solid #E5E5E5` |
| Hover | 背景轻微加深或使用 `#F7F7F7` |
| Focus | `0 0 0 3px rgba(0, 0, 0, 0.12)` |

规则：

- 主要行动每个区域最多一个主按钮。
- 普通操作使用白底描边按钮，保持低噪音。
- 文案保持短动词，例如“搜索”“分析”“收藏”“生成”。

### 卡片

| 类型 | 背景 | 边框 | 圆角 | 阴影 |
| --- | --- | --- | --- | --- |
| 普通卡片 | `#FFFFFF` | `1px solid #E6E8EF` | `12px` | 无 |
| 指标卡片 | `#FFFFFF` | `1px solid #E6E8EF` | `10px` | 无 |
| Chip / 标签 | `#F8FAFC` | `1px solid #EDF0F5` | `999px` | 无 |
| Answer Summary | `#F4F4F4` | 无 | `12px` | 无 |
| Hero 面板 | `linear-gradient(180deg, #FFFFFF 0%, #F7F9FC 100%)` | `1px solid #E6E8EF` | `14px` | 无 |

规则：

- 默认不使用投影，用边框和背景层级区分模块。
- 卡片不要嵌套卡片；需要分组时使用分隔线、标签或表格。
- 列表卡片保持信息密度，标题、描述、元信息三层即可。

### 导航栏 / 侧边栏

| 属性 | 规则 |
| --- | --- |
| 背景 | `#F9F9F9` |
| 边框 | 内容区分隔使用 `1px solid #E5E5E5` |
| 文字 | 默认 `#424242`，选中 `#0D0D0D` |
| 选中态背景 | `#ECECEC` |
| 选中态强调 | 使用深色文字和浅灰背景 |
| 圆角 | 导航项 `9px` 到 `10px` |
| 间距 | 导航项内边距建议 `8px 12px` |

规则：

- 导航保持安静，不使用大面积高饱和色块。
- 当前页面只需要一个清晰选中态，不额外增加阴影。
- 顶部或侧边工具区可使用轻微毛玻璃感，但透明层不应影响文字对比度。

### 输入框和聊天区域

| 属性 | 规则 |
| --- | --- |
| 输入框背景 | `#FFFFFF` |
| 输入框边框 | `1px solid #D9D9D9` |
| 输入框圆角 | `24px` |
| 输入框内边距 | `10px 12px` |
| Placeholder | `#6B6B6B` |
| 聊天气泡圆角 | `18px` |
| 回答摘要背景 | `#F4F4F4` |

规则：

- 聊天输入区优先固定在主流程附近，减少装饰。
- 用户输入和助手回答通过布局、背景或边框区分，不依赖强色块。
- 错误信息使用浅红背景和明确文本，不展示技术栈 traceback。

## 5. 响应式规则

| 断点 | 行为 |
| --- | --- |
| `<= 900px` | 页面左右边距改为 `1rem` |
| `<= 900px` | Stat Grid 从 4 列改为 2 列 |
| `<= 900px` | Hero 标题降为 `1.65rem` |

移动端优先保证表单、按钮和卡片不横向溢出；表格内容必要时使用横向滚动或改为列表展示。
